Introduction

As previously mentioned the mapping of the raw trace data into a unified space is required before it can be effectively analysed and machine learning (ML) applied. In theory you can use ML to define a more effective mapping as a learning goal but, initially at least, you have to map everything into a common representation to achieve this. Doing this with MS2 traces is not trivial as the data is sparse and large, prohibitively large in a uniform representation.

The mapping is crucial. Decisions made at this point impact all downstream analysis. Every technique used here may give a different result if we start with a different initial mapping. In this respect, any conclusion related to say PCA, is not exhaustive but may be specific to PCA given the data involved and the mapping utilised.

In this analysis I look to create the first stage ‘convolutions’ before exploring more transforms and ML models. The raw traces are not uniformly sampled so you first have to decide what uniform space to put them in. Rather than simply super-sampling to a common interpolated space I will look here to create b and y ion optimised mappings.

Here we use the brute force amino combinations up to 9 peptides. There are 659,955 unique masses in the cache file across a range of min 57.02146 to max 1674.71379 mZ. These are used to calculate the possible mZ masses of all unmodified peptides in terms of b and y ions and the raw traces are mapped to only represent these values (currently using a regional maximum). The mass ranges are further limited to ease computation burden to cover a range of around 1,000 mZ.

The analyses are designed to increase in ‘assumptions’ going from unsupervised (i.e. not using peptide information; PCA, robust PCA) to supervised techniques (Partial Least Squares Regression (PLSR), Principal Component Regression (PCR) and Canonical Powered Partial Least Squares (CPPLS) which will optimise based on the specified groupings).

The longer term goal is to derive a useful transformation of the data that we can then use on the MS2 traces in the data files that have not been assigned peptides within the paper so we first seek a task specific result and then will test it on a broader set (possibly not in this analysis scope).

PCA

Simplistically, PCA remaps the the input traces to an alternative space where eigenvalues and eigenvectors have been created to take account of correlations and the algorithm seeks to assign the most variance to vectors in turn. Any original trace can be completely reconstructed by creating the sum of the eigenvectors multiplied by the eigenvalues for that trace. In PCA the eigenvectors are the same size as the original data, i.e. in this analysis 206 (peptides) \(*\) 324,126 mZ values. The data actually gets larger because there also is a matrix of 206 * 206 eigenvalues.

Generally, we will just deal with the reduced dimensional eigenvalues and the hope is that for many systems with correlated variables a reduced number of these dimensions represents important facets of the data.

We can construct a heatmap of eigenvalues. This provides a pictorial representation of the eigenvalues. The heatmap below also has the hierarchical clustering based on the distance matrix for the peptides. In an ideal world all of the peptide groups would cluster together. In my experience, this almost never happens with real world data and PCA. The coloured ribbon on the left hand side shows the group colours for the peptide. There are some blocks of colours so to some degree the PCA and hclust do group certain entries together.

Of note in the heatmap itself is that there are very few obvious blocks of colour and if you squint at the plot youcan see a pattern across the diagonal which suggests that many of the eigenvectors are maximally involved in single peptide traces. This could be to represent noise features or that the MS2 traces are highly distinct.

You can click on the plot to make it bigger (and then right click to open it in another browser tab that will allow you to zoom which makes the text easier to read).

Since the map was optimised for b ions then I would expect clustering to work best for entries starting with the same few amino acids with lesser impact from y ions. This effect working best for the first few masses as beyond that many of the masses are in both b and y ions. I do find it interesting that some of the peptides that start with a different amino acid followed by the same sequence also cluster together in some cases.

We also need to bear in mind that we a viewing a biased set, i.e. peptides found using peaks, we may simply be pulling out things based on algorithm details.

The cumulative variance explained also shows that there is little reduction in the dimension of the data with over 170 eigenvectors required to explain over 99% of the variance. Again this may represent complex differences but also be a factor of noise content.

We can now look at the eigenvalues in terms of the petide groupings. It is common to rescale data before performing PCA (nomalising to the same scale etc.). I prefer not to do this as I like to see mean offsets between traces (though I have perfored a log compression on the data to better match scales). Given this PC1 (i.e. eigenvector 1) in these plots will tend to represent a mean offset value (the actual eigenvectors are shown in the next section). PC1 seems to show little grouping. PC2 is likely to be the first ‘signal’ vector and it seems to relatively neatly split out GVYYPDK group peptides.

Several more eigenvector pairs are presented for visual review.

Hierarchical clustering of PCA eigenvalues and first eigenvectors

The plot below shows the hierarchical clustering plot included in the heatmap followed by some of the first few eigenvector in mZ space. The plots show the sorts of features that the eigenvalue plots above might be emphasising to cluster the groups.

As mentioned in the previous section the first one will be highly weighted towards ‘mean correction’. The second does appear to focus on emphasising and reducing certain masses but raw PCA eigenvectors tend to be hard to interpret as they seek variance and not a more specific class goal. It may take the combination of multiple vectors to clearly show focal areas.

Robust PCA

This section simplistically applies robust PCA to the b ion mappings. This turns out to be computationally expensive and not well tuned (as is usually the case with simply applying algorithms ‘out of the box’). The algorithm eventually decomposed down to a single eigenvector but this section shows the solution at 18 eigenvectors as it is more interesting.

The heatmap of the ‘eigenvalues’ shows much more structure than the PCA plot but actually does not seem to produce a better clustering of the peptides.
